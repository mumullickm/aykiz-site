Privacy Policy

Aykiz is a personal application used by one person: Miraz Uddin Mullick. It is not distributed publicly. The app does not collect data from users, run advertising, or share information with third parties. This document exists so external services (Google, Anthropic) can review how their APIs are used.

What the app accesses

Aykiz reads and writes the following on behalf of the single account holder, only when explicitly enabled in app settings:

  • Google Gmail — read-only access to message metadata (subjects and senders) for up to three most-recent unread messages, surfaced on the Today screen.
  • Google Calendar — read access to today's events, and write access to create or edit events when the user requests it in the assistant chat.
  • Anthropic API — chat messages sent to Claude when the user opens a conversation thread.

Where data lives

  • Conversation threads, journal entries, and project notes are stored locally on the user's device only. No app servers exist.
  • Google data is fetched on demand and never persisted beyond the session view. OAuth tokens are stored in the device's secure storage.
  • Messages sent to Anthropic are handled under Anthropic's own privacy policy at anthropic.com/legal/privacy.

What is never collected

  • No analytics, no crash reporting, no telemetry.
  • No advertising identifiers.
  • No location data.
  • No third-party SDKs that collect data in the background.

Sharing

Nothing is shared with anyone. The app has no backend, no team account, and no analytics provider.

Children

The app is not designed for or distributed to children. It is a single-user adult tool.

Changes

If the policy changes, the change history will be visible in the app's source repository.
